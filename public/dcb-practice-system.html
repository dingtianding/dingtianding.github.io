DCB Practice: system design

Not software a firm uses, the firm itself, delivered as software. Agents run the repetitive tax & accounting work end to end; humans review and approve. A PII-gated ingest and a Postgres system of record make every action auditable, and the firm knowledge base learns from every job.

Legend Practice service Database 👤 Human External data / request learn ↻ · planned
The loop: Agents retrieve how a task should be performed from the firm knowledge base, do the repetitive tax & accounting work using Claude, and hand their output to a PII-gated ingest that rejects unredacted SSN/EIN with a 422 before anything is written. Staff review and approve in the Next.js surface; approved work is recorded to the Postgres system of record, and approved outcomes flow back to continuously learn the knowledge base. DCB Copilot feeds redacted work-context events through the same gate, and the firm's bank transactions (Plaid) are matched to open invoices for a human to confirm. Automation that touches real client work, but every action lands against a record a person can audit.
