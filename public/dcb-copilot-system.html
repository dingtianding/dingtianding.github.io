DCB Copilot — system design

A screen-aware desktop copilot for CPAs. Hit a hotkey and it reads whatever accounting software is on screen, answers in context, and grounds the answer in your firm's documents — entirely on-device, with your own Anthropic key. No server ever sits in the loop.

Legend App window / core On-device store 👤 User External service data BYOK · sync
The loop: a global hotkey wakes the Rust/Tauri core, which grabs an in-memory screenshot of the frontmost accounting app and redacts taxpayer PII. In parallel, firm documents are embedded on-device (BGE-base), stored as a JSON vector index in the app data directory, and reranked (bge-reranker) into cited context. The core sends the redacted screenshot, the question, and that context straight to Claude with your own key — no server of ours in the loop — and the answer streams back into the frameless overlay. A separate Settings window syncs live via Tauri events. Confidentiality by construction: screenshots never touch disk, PII is stripped, and models and the index stay on your machine.
